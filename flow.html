2. REST API Flow
REST API mainly message CRUD ke liye hai. Har successful write operation ke baad WebSocket broadcast bhi hota hai taaki connected clients ko real-time update mile.
📨
POST /api/messages — Message bhejne ka flow
REST se message create karna
▼
Client
HTTP POST request
Client sender aur message body mein bhejta hai.
Router
Validation
sender aur message present hain ya nahi check hota hai. Agar nahi — 400 Bad Request return hota hai.
chatRoutes.js:20
Store
store.addMessage()
In-memory array mein nayi message object push hoti hai. Auto-increment ID milta hai, timestamps set hote hain.
store.js:16
Broadcast
WebSocket broadcast
wsBroadcast({ event: 'new_message', data: newMsg }) call hoti hai — sabhi connected WebSocket clients ko nayi message milti hai.
Response
201 Created
Nayi message object JSON response mein return hoti hai.
Request → Store → Broadcast
// 1. Client se aata hai POST /api/messages { "sender": "Rahul", "message": "Hello!" } // 2. store.js mein save hota hai { id: 1, sender: "Rahul", message: "Hello!", is_edited: false, ... } // 3. WebSocket se sabko broadcast { "event": "new_message", "data": { id: 1, ... } }

3. WebSocket Flow
WebSocket mein client aur server dono directions mein real-time events hote hain — koi polling nahi, persistent connection rehta hai.
🔌
Connection Establish
Client connect hone par kya hota hai
▼
Client
ws://localhost:3020/ws se connect karta hai
TCP handshake + WebSocket upgrade hota hai.
Server — Auto Send 1
history event
Store se sabhi non-deleted messages ek saath client ko bheje jaate hain — taaki chat history dikh sake.
Server — Auto Send 2
online_users event
Currently online users ki list aur count bheja jaata hai.
Client
join event bhejta hai
{ event: "join", data: { name: "Rahul" } } — isse user registered hota hai. Server sabko user_joined broadcast karta hai.
💬
send_message — WebSocket se message bhejne ka flow
Real-time message delivery
▼
// Client A send karta hai { "event": "send_message", "data": { "sender": "Rahul", "message": "Hello!" } } ↓ Server receive karta hai // store.addMessage() → nayi msg banti hai // broadcast() → sabhi clients ko bhejta hai ↓ Client A + B + C sabko milta hai { "event": "new_message", "data": { "id": 1, "sender": "Rahul", "message": "Hello!", ... } }

4. Data Store (store.js)
Store completely in-memory hai — koi database connection nahi. Server restart = sab data gone.
messages[ ]
  • id — auto-increment number
  • sender — username string
  • message — text content
  • message_type — "text" default
  • is_edited — boolean flag
  • is_deleted — soft delete flag
  • created_at / updated_at — ISO timestamps
users[ ] & typing[ ]
  • socketId — IP + timestamp unique key
  • name — display name
  • joined_at — ISO timestamp
  • — — —
  • sender — typing user ka name
  • started_at — typing start time
Store Methods
MethodKaamNotes
getAllMessages() Sabhi non-deleted messages return karta hai is_deleted: false filter lagta hai
getMessageById(id) ID se ek message dhundhta hai Deleted messages nahi milte

updateMessage(id, newMsg) Message content update karta hai is_edited: true set hota hai
deleteMessage(id) Soft delete karta hai is_deleted: true set hota hai
addUser / removeUser Online users manage karta hai Duplicate check hai addUser mein
addTyping / removeTyping Typing users track karta hai Disconnect pe auto-remove hota hai
Complete Lifecycle
═══════════════════════════════════════════════════════ CHAT APP — COMPLETE LIFECYCLE ═══════════════════════════════════════════════════════ [SERVER START] app.js loads → Express configure → HTTP server create → WebSocket attach at /ws → Listen on port 3020 [USER A CONNECTS] ws://localhost:3020/ws → connection event Server → A: { event: "history", data: [...messages] } Server → A: { event: "online_users", data: { count: 1, users: [...] } } [USER A JOINS] A → Server: { event: "join", data: { name: "Rahul" } } store.addUser("Rahul") Server → ALL: { event: "user_joined", data: { name: "Rahul", online_count: 1 } } Server → ALL: { event: "online_users", data: { count: 1, ... } } [USER B CONNECTS & JOINS] Same as above. online_count = 2. [USER A TYPING] A → Server: { event: "typing_start", data: { sender: "Rahul" } } store.addTyping("Rahul") Server → ALL: { event: "typing", data: { sender: "Rahul", is_typing: true, ... } } [USER A SENDS MESSAGE] A → Server: { event: "send_message", data: { sender: "Rahul", message: "Hello!" } } store.addTyping remove → store.addMessage() → id: 5 Server → ALL: { event: "new_message", data: { id: 5, ... } } [USER A EDITS MESSAGE] A → Server: { event: "edit_message", data: { id: 5, message: "Hello World!" } } store.updateMessage(5) → is_edited: true Server → ALL: { event: "message_updated", data: { id: 5, is_edited: true, ... } } [USER A DELETES MESSAGE] A → Server: { event: "delete_message", data: { id: 5 } } store.deleteMessage(5) → is_deleted: true Server → ALL: { event: "message_deleted", data: { id: 5 } } [USER B DISCONNECTS] ws.on('close') fires store.removeUser() + store.removeTyping() Server → ALL: { event: "user_left", data: { name: "Priya", online_count: 1 } } Server → ALL: { event: "online_users", data: { count: 1, ... } }
Example Scenarios
💬 Scenario 1 — REST se message bhejne par WebSocket update
1
Postman se POST /api/messages karo { sender: "Admin", message: "Server message" }
2
Express route mein store.addMessage() call hota hai — message save hoti hai
3
wsBroadcast() call hota hai — sabhi connected WebSocket clients ko new_message event milta hai automatically
4
Postman ko 201 Created milta hai, aur browser chat mein message turant appear ho jaata hai
👥 Scenario 2 — Multiple users ka real-time chat
1
User Rahul aur Priya dono /ws se connect hote hain — dono ko history + online_users milte hain
2
Rahul typing_start bhejta hai → Priya ke screen pe "Rahul is typing..." dikhta hai
3
Rahul send_message karta hai → dono ko new_message broadcast milta hai simultaneously
4
Priya browser close karti hai → Rahul ko user_left event milta hai, online_count: 1
Limitations & Notes
Current Limitations
  • No persistence — restart pe data gone
  • No auth — koi bhi kisi ka naam use kar sakta hai
  • No message limit — unlimited characters
  • No rooms — sirf ek global chat
  • No pagination — sabhi messages ek saath load
Future Improvements
  • PostgreSQL activate karo (schema ready hai)
  • Auth/JWT add karo
  • Message length limit add karo
  • Multiple rooms support
  • Pagination for history
