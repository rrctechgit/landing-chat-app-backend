Chat App API

Complete REST API + WebSocket documentation for the landing-chat-app backend. All endpoints, request payloads, response structures, and real-time event schemas.

Port 3020
Runtime Node.js / Express
DB PostgreSQL
Auth None
Format JSON
Base URL
🌐 http://localhost:3020
🔌 ws://localhost:3020/ws
All REST endpoints are prefixed with /api. No authentication required on any endpoint.
Standard Error Format

All error responses follow this structure:

Error Response
{ "success": false, "error": "Human-readable error message" }
REST Endpoints
GET /api/messages Fetch all messages ▼
Returns all non-deleted messages from the in-memory store, ordered by creation time.
Request
// No request body or query parameters needed GET /api/messages HTTP/1.1 Host: localhost:3020
200 OK   Success Response
{ "success": true, "data": [ { "id": 1, "sender": "john_doe", "message": "Hello everyone!", "message_type": "text", "is_edited": false, "is_deleted": false, "created_at": "2026-06-18T10:30:00.000Z", "updated_at": "2026-06-18T10:30:00.000Z" } ] }
GET /api/messages/:id Fetch single message ▼
URL Parameters
ParameterTypeRequiredDescription
id number Required Numeric ID of the message
Request
GET /api/messages/1 HTTP/1.1 Host: localhost:3020
200 OK   Success Response
{ "success": true, "data": { "id": 1, "sender": "john_doe", "message": "Hello everyone!", "message_type": "text", "is_edited": false, "is_deleted": false, "created_at": "2026-06-18T10:30:00.000Z", "updated_at": "2026-06-18T10:30:00.000Z" } }
404 Not Found
{ "success": false, "error": "Message not found" }
POST /api/messages Create a new message ▼
After creating, broadcasts a new_message WebSocket event to all connected clients.
Request Body  Content-Type: application/json
FieldTypeRequiredDefaultDescription
sender string Required — Username of the message sender
message string Required — Text content of the message
message_type string Optional "text" Type of message (e.g. text, image)
Request Payload
POST /api/messages HTTP/1.1 Host: localhost:3020 Content-Type: application/json { "sender": "john_doe", "message": "Hello everyone!", "message_type": "text" // optional }
201 Created   Success Response
{ "success": true, "data": { "id": 5, "sender": "john_doe", "message": "Hello everyone!", "message_type": "text", "is_edited": false, "is_deleted": false, "created_at": "2026-06-18T10:30:00.000Z", "updated_at": "2026-06-18T10:30:00.000Z" } }
400 Bad Request
{ "success": false, "error": "sender and message are required" }
PUT /api/messages/:id Edit a message ▼
Sets is_edited: true and updates updated_at. Broadcasts message_updated WebSocket event.
URL Parameters
ParameterTypeRequiredDescription
id number Required Numeric ID of the message to update
Request Body
FieldTypeRequiredDescription
message string Required New text content for the message
Request Payload
PUT /api/messages/5 HTTP/1.1 Host: localhost:3020 Content-Type: application/json { "message": "Updated message content" }
200 OK
{ "success": true, "data": { "id": 5, "sender": "john_doe", "message": "Updated message content", "message_type": "text", "is_edited": true, "is_deleted": false, "created_at": "2026-06-18T10:30:00.000Z", "updated_at": "2026-06-18T10:45:00.000Z" } }
400 Bad Request
{ "success": false, "error": "message is required" }
404 Not Found
{ "success": false, "error": "Message not found" }
DEL /api/messages/:id Soft-delete a message ▼
This is a soft delete — sets is_deleted: true. The record is not removed from the database. Broadcasts message_deleted WebSocket event.
URL Parameters
ParameterTypeRequiredDescription
id number Required Numeric ID of the message to delete
Request
DELETE /api/messages/5 HTTP/1.1 Host: localhost:3020 // No request body needed
200 OK
{ "success": true, "message": "Message deleted", "id": 5 }
404 Not Found
{ "success": false, "error": "Message not found" }

WebSocket

The server uses the native ws library. All messages are JSON strings.

🔌 ws://localhost:3020/ws
On successful connection, the server automatically sends a history event with all existing messages, and an online_users event.
General Message Format
{ "event": "event_name", "data": { /* event-specific payload */ } }
WebSocket — Client → Server Events
Client → Server join Register a username ▼
Payload Fields
FieldTypeRequiredDescription
name string Required Display name for the user
Send Payload
{ "event": "join", "data": { "name": "john_doe" } }
Triggers broadcast of user_joined and updated online_users to all clients.
Client → Server send_message Post a message ▼
Payload Fields
FieldTypeRequiredDefaultDescription
sender string Required — Sender's username
message string Required — Message text content
message_type string Optional "text" Type of message
Send Payload
{ "event": "send_message", "data": { "sender": "john_doe", "message": "Hey, how are you?", "message_type": "text" } }
Client → Server edit_message Edit an existing message ▼
Payload Fields
FieldTypeRequiredDescription
id number Required ID of the message to edit
message string Required New message content
Send Payload
{ "event": "edit_message", "data": { "id": 5, "message": "Corrected message" } }
Client → Server delete_message Delete a message ▼
Payload Fields
FieldTypeRequiredDescription
id number Required ID of the message to delete
Send Payload
{ "event": "delete_message", "data": { "id": 5 } }
Client → Server typing_start User started typing ▼
Send Payload
{ "event": "typing_start", "data": { "sender": "john_doe" } }
Broadcasts a typing event with is_typing: true to all other clients.
Client → Server typing_stop User stopped typing ▼
Send Payload
{ "event": "typing_stop", "data": { "sender": "john_doe" } }
Broadcasts a typing event with is_typing: false to all other clients.

Database Schema
Currently the server stores data in-memory via src/store.js. The PostgreSQL table is defined but not yet active — data resets on server restart.
🗄 Table: chatapp
Column Type Nullable Default Notes
id SERIAL NOT NULL auto-increment Primary Key
room VARCHAR(100) NOT NULL "general" Chat room name
sender VARCHAR(100) NOT NULL — Username
message TEXT NOT NULL — Message body
message_type VARCHAR(20) NOT NULL "text" e.g. text, image
is_edited BOOLEAN NOT NULL FALSE Edit flag
is_deleted BOOLEAN NOT NULL FALSE Soft delete flag
created_at TIMESTAMPTZ NOT NULL NOW() Creation timestamp
updated_at TIMESTAMPTZ NOT NULL NOW() Last update timestamp
Create Table SQL
CREATE TABLE chatapp ( id SERIAL PRIMARY KEY, room VARCHAR(100) NOT NULL DEFAULT 'general', sender VARCHAR(100) NOT NULL, message TEXT NOT NULL, message_type VARCHAR(20) NOT NULL DEFAULT 'text', is_edited BOOLEAN NOT NULL DEFAULT FALSE, is_deleted BOOLEAN NOT NULL DEFAULT FALSE, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW() );
